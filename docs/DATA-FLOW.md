# 데이터 흐름 및 핵심 규칙

## 데이터 흐름 3단계

```
[입력] User → DB 실시간 저장
[계산] Admin 마감 클릭 → 서버 계산 → Snapshot 저장
[조회] User/Admin → Snapshot 기반 조회
```

## 주간 마감 프로세스 (Close Week)

Admin이 `[Close Week N]` 버튼 클릭 시:

```
1. 개인 달성률 계산
   - 각 유저의 목표 대비 현재 수치 → 달성률 (0~100%)
   - 누적 기준, 100% cap

2. 입력 누락자 처리
   - 해당 주 인바디 미입력 → 직전 주 데이터 그대로 유지

3. 팀 점수 계산
   - 팀원 2명의 달성률 평균

4. 팀 순위 산정
   - 해당 주 팀 점수 기준 내림차순

5. 하위 2팀 확정

6. 결과 스냅샷 저장
   - WeeklySnapshot 테이블에 저장
   - 저장 후 수정 불가 (immutable)
```

## 최종 순위 계산 (시즌 종료)

```
FinalScore = Week1 점수 + Week2 점수 + Week3 점수 + Week4 점수
최종 순위 = FinalScore 내림차순
```

## 핵심 규칙

| 규칙 | 설명 |
|------|------|
| Snapshot 불변 | 주간 마감 후 데이터는 절대 수정 불가 |
| 누락 = 유지 | 입력 안 하면 직전 주 값 그대로 사용 |
| 주간 리셋 | 매주 순위는 독립 계산 |
| 100% Cap | 달성률은 최대 100% |
| 팀 미션 1개 | 주당 팀 단위 미션 1개만 |
